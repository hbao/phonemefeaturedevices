Introduction

phoneME™ Feature is an open source version of the Sun Java™ Wireless Client Software product from Sun Microsystems, an optimized Java platform, Micro Edition (Java ME) stack. phoneME Feature supports multitasking and is built upon MIDP 2.1. For more information about phoneME Feature, see Software Overview.

The following features are part of phoneME Feature:

  • New support for Advanced Multitmedia Supplements (JSR 234)
  • New support for Mobile Sensors API (JSR 256)
  • New support for XML API for Java ME (JSR 280)
  • Addressed MR2 audio encumbrance with addition of a sound engine binary plug-in
  • Ongoing support for the following Java Specification Requests (JSRs):
    • File Connection and Personal Information Management API (JSR 75)
    • Bluetooth (JSR 82)
    • Mobile Information Device Profile (JSR 118)
    • Wireless Messaging API 1.0 (JSR 120)
    • Mobile Media API (JSR 135)
    • Webservices API (JSR 172)
    • Security and Trust Services API (JSR 177)
    • Location API (JSR 179)
    • Session Initiation Protocol (JSR 180)
    • Wireless Messaging API 2.0 (JSR 205)
    • Content Handler API (JSR 211)
    • Scalable 2D Vector Graphics (JSR 226)
    • Payment API (JSR 229)
    • Mobile Internationalization (JSR 238)
    • Java Bindings for OpenGL™ Embedded Subset (JSR 239)
  • Support for JavaCall™ porting interfaces on Win32 platform
  • Enhancements to the Application Management System (AMS)
  • Enhanced logging capabilities
  • Enhanced on-device debugging capabilities
  • Footprint and performance is maintained with added features since the MR2 release

Implementation Notes

Support for Linux on ARM Platform

phoneME Feature MR3 software supports building on Linux for the ARM target platform. However, because embedded platforms such as Linux on ARM tend to be very specific, phoneME Feature MR3 has been ported to the Texas Instruments P2SAMPLE64-V6 board as a sample ARM platform. It does not consitute a fully-qualified port, but is meant to serve as a starting point for Linux on ARM platforms in general.

For more information on building phoneME Feature for the Linux on ARM platform, see the Sun Java Wireless Client Software Build Guide.

MT-QTS No Longer Supported

Support for the Multitasking Quality Test Suite (MT-QTS) has been dropped in phoneME Feature. Future support for Multitasking Test Suite will be made available in the next release of the Java Device Test Suite (JDTS) from Sun Microsystems.

Quick Key Presses in WTK 2.5.2 Can Lose Key Strokes

phoneME Feature is dependent on components of Wireless Toolkit (WTK) 2.5.2 for emulation on Win32 platforms. In phoneME Feature builds on Win32 platforms with WTK 2.5.2, very quick, repeated keypresses from QWERTY keyboards can be lost. To bypass the problem, press repeated keys with a pause of 500ms or more. Users should be careful typing passwords and URLs. (See bug 6682099 in Known Bugs, below.)

JAVA_HEAP_SIZE Property Value Incorrect

MIDlets installed in SVM mode are unable to be run twice, with no error messages appearing in the console. The problem is caused by an incorrect value in JAVA_HEAP_SIZE in the jwc_properties.ini file. (See bug 6680850 in Known Bugs, below.)

Source Root Directory Path Length May Cause Unpredictable Build Crashes

Due to an inconsistency in Cyg4me, phoneME Feature source root directory path names that are too long or too short may (or may not) cause make to crash during building, with the following error message:

    4 [main] sh 1560 handle_exceptions: Exception: STATUS_ACCESS_VIOLATION
    653 [main] sh 1560 open_stackdumpfile: Dumping stack trace to sh.exe.stackdump
    make: *** [xxxxxxxxxxxxxxxxxxxxxxxxxxxx] Error 10112

There are no strict rules for how long or how short a source root directory path must be to successfully sidestep this bug. If it occurs, work around it by shortening or lengthening your source root directory path names. (See bug 6393610 in Known Bugs, below.)
